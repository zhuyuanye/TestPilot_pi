# 用户管理验收标准

## 范围说明

- 本文仅覆盖 `docs/requirements/user-management.md` 和产品澄清 1～8 中已明确的规则。
- 按当前决策，尚未确认的问题不阻塞本版 AC，也不通过假设补充规则。
- 删除只验证用户可观察的查询结果，不直接验证数据库物理记录。（来源：产品澄清 6）
- 新增成功只验证用户能够从用户列表中查询到，不验证用户能否立即登录。（来源：产品澄清 5）
- 失败信息无需逐字匹配，但应说明对应的失败原因。（来源：产品澄清 8）

## 验收标准

### AC-01 有效信息可以新增用户

**来源：** 原始需求 §1“业务背景”、§2“新增用户”

- **Given** 系统管理员提供的用户名、昵称、密码和可选手机号均符合已确认规则
- **And** 用户名在系统中不存在
- **When** 系统管理员新增该用户
- **Then** 新增操作成功

### AC-02 用户名必填

**来源：** 原始需求 §2“新增用户”；产品澄清 2、8

- **Given** 系统管理员未提供用户名，其他信息均符合已确认规则
- **When** 系统管理员新增用户
- **Then** 新增操作失败，且失败信息说明用户名为必填项

### AC-03 用户名达到最小长度时可以新增

**来源：** 原始需求 §2“新增用户”；产品澄清 2

- **Given** 系统管理员提供一个长度为 2 个字符的非空且唯一的用户名
- **And** 其他信息均符合已确认规则
- **When** 系统管理员新增用户
- **Then** 新增操作成功

### AC-04 用户名低于最小长度时新增失败

**来源：** 原始需求 §2“新增用户”；产品澄清 2、8

- **Given** 系统管理员提供一个长度为 1 个字符的用户名
- **And** 其他信息均符合已确认规则
- **When** 系统管理员新增用户
- **Then** 新增操作失败，且失败信息说明用户名长度不符合要求

### AC-05 用户名达到最大长度时可以新增

**来源：** 原始需求 §2“新增用户”；产品澄清 2

- **Given** 系统管理员提供一个长度为 20 个字符的非空且唯一的用户名
- **And** 其他信息均符合已确认规则
- **When** 系统管理员新增用户
- **Then** 新增操作成功

### AC-06 用户名超过最大长度时新增失败

**来源：** 原始需求 §2“新增用户”；产品澄清 2、8

- **Given** 系统管理员提供一个长度为 21 个字符的用户名
- **And** 其他信息均符合已确认规则
- **When** 系统管理员新增用户
- **Then** 新增操作失败，且失败信息说明用户名长度不符合要求

### AC-07 用户名不限制字符类型

**来源：** 产品澄清 2

- **Given** 用户名非空、唯一且长度为 2～20 个字符
- **And** 其他信息均符合已确认规则
- **When** 系统管理员使用该用户名新增用户
- **Then** 新增操作成功

### AC-08 用户名唯一性不区分大小写

**来源：** 原始需求 §2“新增用户”；产品澄清 1、8

- **Given** 系统中已存在一个用户
- **And** 按不区分大小写的方式比较时，待新增用户名与该用户的用户名相同
- **When** 系统管理员使用待新增用户名新增用户
- **Then** 新增操作失败，且失败信息说明用户名已存在

### AC-09 用户昵称必填

**来源：** 原始需求 §2“新增用户”；产品澄清 8

- **Given** 系统管理员未提供用户昵称，其他信息均符合已确认规则
- **When** 系统管理员新增用户
- **Then** 新增操作失败，且失败信息说明用户昵称为必填项

### AC-10 密码必填

**来源：** 原始需求 §2“新增用户”；产品澄清 4、8

- **When** 系统管理员新增用户
- **Then** 新增操作失败，且失败信息说明密码为必填项

### AC-11 密码达到最小长度时可以新增

**来源：** 产品澄清 4

- **And** 其他信息均符合已确认规则
- **When** 系统管理员新增用户
- **Then** 新增操作成功

### AC-12 密码低于最小长度时新增失败

**来源：** 产品澄清 4、8

- **Given** 系统管理员提供一个长度为 4 个字符的密码
- **And** 其他信息均符合已确认规则
- **When** 系统管理员新增用户
- **Then** 新增操作失败，且失败信息说明密码长度不符合要求

### AC-13 密码达到最大长度时可以新增

**来源：** 产品澄清 4

- **Given** 系统管理员提供一个长度为 20 个字符的密码
- **And** 其他信息均符合已确认规则
- **When** 系统管理员新增用户
- **Then** 新增操作成功

### AC-14 密码超过最大长度时新增失败

**来源：** 产品澄清 4、8

- **Given** 系统管理员提供一个长度为 21 个字符的密码
- **And** 其他信息均符合已确认规则
- **When** 系统管理员新增用户
- **Then** 新增操作失败，且失败信息说明密码长度不符合要求

### AC-15 密码不要求字符复杂度

**来源：** 产品澄清 4

- **Given** 密码长度为 5～20 个字符但不包含多种字符类型
- **And** 其他信息均符合已确认规则
- **When** 系统管理员使用该密码新增用户
- **Then** 新增操作成功

### AC-16 手机号可以不填写

**来源：** 原始需求 §2“新增用户”

- **Given** 系统管理员未填写手机号
- **And** 其他信息均符合已确认规则
- **When** 系统管理员新增用户
- **Then** 新增操作成功

### AC-17 符合格式的手机号可以用于新增

**来源：** 原始需求 §2“新增用户”；产品澄清 3

- **Given** 系统管理员填写一个由 11 位数字组成且第一位为数字 1 的手机号
- **And** 其他信息均符合已确认规则
- **When** 系统管理员新增用户
- **Then** 新增操作成功

### AC-18 不符合格式的手机号导致新增失败

**来源：** 原始需求 §2“新增用户”；产品澄清 3、8

- **Given** 系统管理员填写的手机号不是由 11 位数字组成，或者第一位不是数字 1
- **And** 其他信息均符合已确认规则
- **When** 系统管理员新增用户
- **Then** 新增操作失败，且失败信息说明手机号格式不符合要求

### AC-19 新增成功后可以从用户列表查询到用户

**来源：** 原始需求 §2“新增用户”；产品澄清 5

- **Given** 系统管理员已成功新增一个用户
- **When** 系统管理员在用户列表中查询该用户
- **Then** 能够查询到该用户

### AC-20 新增用户的默认状态为正常

**来源：** 原始需求 §2“新增用户”

- **Given** 系统管理员新增用户时未指定其他状态
- **When** 用户新增成功
- **Then** 该用户的状态为“正常”

### AC-21 管理员可以删除普通系统用户

**来源：** 原始需求 §3“删除用户”

- **Given** 管理员已登录
- **And** 目标用户是普通系统用户且不是当前登录用户
- **When** 管理员删除目标用户
- **Then** 删除操作成功

### AC-22 删除成功后用户不再出现在用户列表中

**来源：** 原始需求 §3“删除用户”；产品澄清 6、7

- **Given** 一个用户已被成功删除
- **When** 管理员在用户列表中查询该用户
- **Then** 不能查询到该用户

### AC-23 删除成功后不能再查询用户详情

**来源：** 产品澄清 6、7

- **Given** 一个用户已被成功删除
- **When** 管理员查询该用户的详情
- **Then** 不能查询到该用户

### AC-24 当前登录用户不能删除自己

**来源：** 原始需求 §3“删除用户”；产品澄清 8

- **Given** 管理员已登录
- **When** 管理员尝试删除当前登录用户
- **Then** 删除操作失败，当前登录用户仍然可以被查询到，且失败信息说明不能删除自己

### AC-25 删除不存在的用户不能影响其他用户

**来源：** 原始需求 §3“删除用户”

- **Given** 目标用户不存在，且系统中存在其他用户
- **When** 管理员尝试删除该目标用户
- **Then** 其他用户仍然可以被查询到，且其用户信息保持不变

## 最终评审结论

| 检查项 | 结论 |
|---|---|
| 可追溯性 | 通过。每条 AC 均标注原始需求章节或产品澄清编号，追踪矩阵覆盖全部原始规则和 8 项产品澄清。 |
| 可测试性 | 通过。正常、异常和边界结果均可通过操作状态及用户查询结果观察；失败信息按“说明对应失败原因”判定，不要求逐字匹配。 |
| 重复或遗漏 | 通过。相同规则的正常、异常和边界场景分别表达；新增、删除、权限和数据一致性规则均已覆盖。 |
| 主观或模糊预期 | 已处理。除产品明确不要求逐字匹配的失败文案外，AC 使用成功、失败、字段长度、状态和查询结果等可观察结果。 |
| 实现细节 | 通过。未引入接口路径、数据库字段、物理删除方式或登录机制；用户详情查询仅来源于产品澄清 7。 |
| 未决事项 | 按当前决策不纳入本版 AC，也未通过假设补充相关规则。 |
