# 用户管理缺陷清单

## 执行环境

- RuoYi-Vue 3.9.2
- 后端：`http://localhost:8080`
- 前端：`http://localhost:8081`
- 依据：`outputs/user-management-acceptance-criteria.md`
- 详细结果：`outputs/test-execution-report.md`

## 动态验证缺陷

| 缺陷 | 严重级别 | AC / TC | 实际结果 | 建议 |
|---|---|---|---|---|
| BUG-001 后端用户名长度规则不符合需求 | 高 | AC-04、AC-06 / TC-004 | 1 位和 21 位用户名均新增成功 | 后端统一校验 2～20 位，不能只依赖 Vue |
| BUG-002 昵称必填校验过晚且泄露内部信息 | 高 | AC-09 / TC-008 | 空昵称进入数据库后失败，响应暴露 SQL、Mapper 路径 | 在 API 参数层校验必填，返回稳定业务错误 |
| BUG-003 后端缺少密码必填及长度校验 | 高 | AC-10、AC-12、AC-14 / TC-009、TC-010 | 空密码、4 位和 21 位密码均新增成功 | 后端增加必填及 5～20 位校验 |
| BUG-004 后端手机号格式校验不完整 | 中 | AC-18 / TC-013 | 10 位、非 1 开头、含非数字手机号均新增成功 | 后端按确认口径验证 11 位数字且以 1 开头 |
| BUG-005 已删除用户仍可通过详情接口查询 | 高 | AC-23 / TC-015 | 列表不可见，但详情接口仍返回逻辑删除用户 | `selectUserById` 增加删除标记过滤 |

## 静态分析待确认项

| 风险 | 证据 | 处理建议 |
|---|---|---|
| Vue 密码最小长度为 6，产品澄清为 5 | `ruoyi-ui/src/utils/passwordRule.js` | 补充 5 位密码 UI 用例，确认后登记独立缺陷 |
| Vue 手机号要求第二位 3～9，产品只要求首位为 1 | `ruoyi-ui/src/views/system/user/index.vue` | 明确产品口径是否需要收紧，再决定修改需求或前端 |
| 用户名大小写唯一性依赖数据库排序规则 | Mapper 使用 `=`，当前环境测试通过 | 在应用或数据库层显式保证并增加回归测试 |

## 结论

当前核心 UI 流程可完成，但 API 验收存在 5 组已复现缺陷，用户管理需求不能判定为全部通过。
