# 测试流程兜底规范

兜底用于模型、网络、工具链或演示时间异常，不能冒充现场生成。

## 启用条件

- 单阶段超过已声明时间预算；
- 模型或网络失败；
- 本地工具链无法及时恢复；
- 与产品行为无关的环境故障。

## 必须执行

1. 说明受阻阶段、原因和影响。
2. 获得人工明确批准。
3. 向 `<workspace>/FALLBACK_USED.md` 追加时间、阶段、原因、来源版本、复制文件和批准说明。
4. 排除 `.env`、凭据、构建目录、历史日志、XML、Trace、截图、录像和历史执行报告。
5. 当前环境重新执行复制来的自动化。
6. 最终报告明确标记兜底来源。

## 禁止行为

- 需求分析前静默读取兜底 AC、用例或代码；
- 单阶段受阻却替换整个流程；
- 把历史绿色结果写成本次结果；
- 删除或覆盖兜底记录；
- 对观众声称兜底材料是本次现场生成。
