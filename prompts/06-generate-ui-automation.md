# Prompt 06：生成并执行 Vue UI 自动化

```text
请基于 outputs 中已确认的 AC、测试用例和实现分析，实现 TC-001 的 UI 自动化。

目标：通过真实 Vue 页面完成管理员登录、新增用户、搜索确认和删除用户。

先分析现有 automation/ui；如果目录不存在则创建独立 Maven 工程。先输出实施计划，等待我确认后再修改代码。

约束：
1. 使用 Java 17、JUnit 5、Playwright Java、Chromium
2. 不修改 RuoYi-Vue 业务代码
3. UI 地址、API 地址和账号凭据通过环境变量传入
4. 支持 HEADLESS 和 SLOW_MO，现场可见运行
5. 使用用户可感知属性或稳定作用域定位，不使用脆弱的全局 nth 定位
6. 动态生成唯一用户名和合法手机号
7. 验证新增成功、列表中存在、删除成功及列表中消失
8. finally/AfterEach 中通过 API 兜底清理数据
9. 失败时保存截图，但日志和截图命名不得包含密码或 Token
10. 测试标注 TC-001 与 AC-01、AC-19、AC-20、AC-21、AC-22

实现后：
- 安装或确认 Chromium
- 先无头执行验证稳定性
- 再给出现场有头慢动作执行命令
- 输出修改文件、执行结果、覆盖范围和剩余风险
```
