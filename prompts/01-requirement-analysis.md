# Prompt 01：需求分析

```text
你是一名资深测试工程师。

请只阅读 docs/requirements/user-management.md 中的原始需求。
本阶段不要查看业务实现，不要读取 presenter 目录，不要生成 AC、测试用例或自动化代码。

1. 从需求中提取的明确业务规则
2. 需求中的歧义、缺失信息和待确认问题
3. 可能影响测试设计的业务风险
4. 建议向产品经理确认的问题，并按优先级排序
5. 当前可以直接转化为 AC 的规则
6. 必须澄清后才能转化为 AC 的规则

约束：
- 不得自行补充需求中未说明的规则
- 推测必须标记为“假设”
- 每项结论引用对应的需求章节
- 本阶段只在对话中输出分析，不修改任何文件
```
