# Prompt 01：现场需求分析（只读原始需求）

> 现场输入本提示词。不要先打开 `outputs/`、`automation/` 或 `presenter/checkpoints/`。

```text
你是一名资深测试工程师。我们正在进行现场演示，本轮所有新产物必须放在 demo-live/，仓库现有 outputs/、automation/ 和 presenter/checkpoints/ 只允许在我明确说“启用兜底”后读取。

请只阅读 docs/requirements/user-management.md。
本阶段禁止查看业务实现，禁止读取 presenter 目录，禁止读取已有 AC、用例、报告和自动化代码。

请在对话中输出：
1. 从原始需求提取的明确业务规则，并引用章节
2. 歧义、缺失信息和冲突
3. 按“阻塞验收 / 重要 / 可后续确认”分级的问题
4. 建议向产品经理提出的问题
5. 当前可直接转成 AC 的规则
6. 必须澄清后才能转成 AC 的规则

约束：
- 不得自行补充需求中没有的规则
- 推测必须显式标记为“假设”
- 此时不要生成 AC、测试用例或代码
- 不修改任何文件
- 输出后停止，等待我扮演产品角色逐项回复
```

## 人工控制点

现场测试人员检查 Agent 是否真的发现必填项、长度、唯一性、手机号、删除一致性等未决口径。若遗漏，直接追问，不要展示预置答案。

产品回复完成后输入：

```text
请基于刚才的原始需求分析和产品回复，列出“已澄清 / 仍未决 / 冲突”三类结论。不要生成 AC。经我确认后，将评审通过的分析写入 demo-live/outputs/01-requirement-analysis.md，然后停止。
```
